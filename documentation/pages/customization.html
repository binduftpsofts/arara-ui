Customization

ArarA is designed to be fully extensible. By leveraging Sass's modular @use system, you can override every variable, add your own utilities, and use powerful mixins to build a truly unique design system.

Sass Architecture

ArarA follows a logical 4-layer structure to keep code organized and maintainable.

1. Abstracts (The Brain)

Central variables, functions, and mixins. No CSS is output from this layer.

2. Base (The Foundation)

Browser resets and global typography defaults.

3. Components (UI Elements)

Optional higher-level patterns like buttons, cards, and navigation items.

4. Utilities (The Tools)

Atomic classes generated from variables. Loaded last for maximum specificity.

Overriding Variables

To customize the theme, simply define your own variables before importing ArarA's modules. All internal variables use the !default flag.

scss
// your-styles.scss
  
  // 1. Override the primary color
  $colors: (
    "primary": #ff4500, // Custom Orange
    "slate-500": #555555
  );
  
  // 2. Load ArarA
  @use 'path/to/arara/src/scss/main';

Using Mixins

Use ArarA's internal logic in your own Sass files for consistency.

respond-to($breakpoint)

Generate mobile-first media queries effortlessly.

css
.my-custom-card {
  width: 100%;
  
  @include respond-to('md') {
    width: 50%;
  }
}

glassify($bg, $blur)

Apply complex glassmorphism effects to any custom component.

css
.premium-header {
  @include glassify(rgba(255, 255, 255, 0.2), 15px);
}

generate-utility($prefix, $property, $map)

The core engine used to build ArarA. Create your own atomic utilities from a Sass map.

css
$my-sizes: ("tiny": 4px, "huge": 100px);

@include generate-utility("custom", "padding", $my-sizes);

// Generates:
// .custom-tiny { padding: 4px !important; }
// .custom-huge { padding: 100px !important; }
