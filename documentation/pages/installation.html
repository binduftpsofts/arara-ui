Installation

Get ArarA into your project in minutes. Choose the method that fits your workflow CDN for instant access, NPM for a production pipeline, or Sass for full customization power.

Via CDN

The fastest way to get started. Add the stylesheet link to your HTML <head> — no build step required.

HTML
<link rel="stylesheet" href="https://cdn.arara-css.com/v0.1.0/style.css">

Via NPM

Install ArarA as a development dependency and import the compiled stylesheet.

bash
npm install arara-css

Then link the compiled file in your HTML:

HTML
<link rel="stylesheet" href="node_modules/arara-css/dist/css/style.css">

Local Sass Build

For full control over the framework — override variables, add utilities, or cherry-pick modules — clone the repository and build locally.

1. Clone and install

HTML
git clone https://github.com/FazleRabbiBindu/arara.git
cd arara
npm install

2. Build the CSS

HTML
# One-time production build
npm run build

# Watch mode (auto-recompile on save)
npm run watch

3. Link in your HTML

HTML
<link rel="stylesheet" href="dist/css/style.css">

Project Structure

After cloning, you'll find this directory structure:

HTML
arara/
├── src/scss/
│   ├── main.scss              ← Entry point
│   ├── abstracts/             ← Functions, Mixins, Animations
│   ├── base/                  ← Reset, Global Styles
│   ├── components/            ← TOC, Drawer, Chapter Nav
│   ├── utilities/             ← Spacing, Colors, Grid, Visuals
│   └── variables/             ← Design Tokens (colors, spacing…)
├── dist/css/
│   └── style.css              ← Compiled output
├── documentation/             ← This documentation site
│   ├── pages/
│   └── scripts/
└── index.html                 ← Documentation homepage
