Color Utilities SCSS

A complete, atomic CSS utility generator for colors. For every key in your $colors map, the system automatically creates text, background, border, hover, and opacity variants — with zero redundancy.

How It Works

The SCSS loops through a $colors map and generates every utility class dynamically. No manual CSS writing – just configure your palette once.

scss
$colors: (
  'primary':   #3b82f6,
  'secondary': #64748b,
  'danger':    #ef4444,
  'success':   #22c55e,
  'warning':   #eab308,
  'info':      #0ea5e9,
  'dark':      #0f172a,
  'light':     #f8fafc
);

Each key becomes a class suffix – e.g., .text-primary, .bg-danger.
The code relies on a str-replace function to handle slash‑based opacity classes (like .bg-primary/20).

Configuration & Dependencies

Required str-replace function

Opacity classes (.bg-primary/20) need a string replace function to escape the slash. Add this to your abstracts:

scss
@function str-replace($string, $search, $replace: '') {
  $index: str-index($string, $search);
  @if $index {
    @return str-slice($string, 1, $index - 1) + $replace + 
           str-replace(str-slice($string, $index + str-length($search)), $search, $replace);
  }
  @return $string;
}

Customizing the palette

Add any color (hex, rgb, hsl, named) to $colors and recompile. All utilities regenerate automatically.

scss
$colors: (
  'brand': #ff4785,
  'accent': #00c6a7,
  'custom': #b94bff
);
